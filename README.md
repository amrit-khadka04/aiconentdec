# FrameScope — AI Video Content Detector

FrameScope analyses uploaded video files frame-by-frame and tells you whether the content was generated by AI or captured by a real camera.  
It combines a fine-tuned **EfficientNet-B4** deepfake-detection model with a **Groq LLM** that writes a plain-English explanation of every suspicious frame.

## How It Works

```
Video Upload
     │
     ▼
Frame Extraction  ──►  ML Detector (EfficientNet-B4)  ──►  AI Score per Frame
                                                                    │
                        Forensic Signal Extraction ─────────────────┘
                        (texture / noise / frequency / colour)
                                    │
                                    ▼
                        Groq LLM Explanation (per frame)
                                    │
                                    ▼
                        Ensemble — Overall Verdict
                        (ai / uncertain / human)
```

1. **Frame Extraction** — OpenCV samples the video at a configurable rate (default every 30 frames, up to 40 frames total).
2. **ML Detection** — Each frame is run through [`dima806/deepfake_vs_real_image_detection`](https://huggingface.co/dima806/deepfake_vs_real_image_detection) (EfficientNet-B4 trained on 190 k real/fake images) and gets an AI probability score (0 = human, 1 = AI).
3. **Signal Extraction** — Multiple complementary forensic signals are computed per frame (texture/noise/frequency/colour + contrast, gradient, spectral, saturation, and regional consistency checks).
4. **LLM Explanation** — The Groq API sends the frame image + score + signals to a vision-capable model which returns a natural-language forensic report.
5. **Ensemble** — A video-level weighted fusion is used (mean ML score + forensic signals + top-quartile ML + temporal evidence from coverage/peak/persistence):
   - `>= 0.50` → **AI**
   - `0.30 – 0.50` → **Uncertain**
   - `< 0.30` → **Human**

## Backend Setup

### 1. Create and activate a virtual environment

```bash
cd framescope/backend
python -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

### 2. Install Python dependencies

```bash
pip install -r requirements.txt
```

> **Note:** `torch` (~2 GB) and the HuggingFace model (~400 MB) are downloaded on first run and cached locally.  
> On CPU-only machines this may take a few minutes.

### 3. Create the environment file

Create a file called `.env` inside the `framescope/` directory (one level above `backend/`):

```bash
# framescope/.env
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

Replace the value with your real Groq API key from [console.groq.com/keys](https://console.groq.com/keys).

Optional settings you can also add to `.env`:

```ini
# Model used for frame explanations (default shown)
GROQ_MODEL=meta-llama/llama-4-scout-17b-16e-instruct

# How many video frames to skip between samples (default 30)
DEFAULT_SAMPLE_RATE=30

# Maximum number of frames to analyse per video (default 40)
DEFAULT_MAX_FRAMES=40
```

## Running the Application

Open **two terminal windows** (or tabs).

### Terminal 1 — Backend

```bash
cd framescope/backend
source .venv/bin/activate   # Windows: .venv\Scripts\Activate.ps1
uvicorn main:app --reload --port 8000
```

Expected output:

```
Loading ML model...
Model labels discovered: [{'label': 'FAKE', 'score': ...}, ...]
Model ready.
INFO:     Uvicorn running on http://127.0.0.1:8000
```

> The first launch downloads the HuggingFace model. Subsequent launches use the local cache.

### Terminal 2 — Frontend

```bash
cd framescope/frontend
npm run dev
```

Expected output:

```
  VITE v5.x.x  ready in XXX ms

  ➜  Local:   http://localhost:5173/
```

Open **http://localhost:5173** in your browser.

## Using FrameScope

1. **Upload a video** — Drag and drop (or click to browse) a video file.  
   Supported formats: `.mp4`, `.mov`, `.avi`, `.webm`, `.mkv`.
2. **Wait for analysis** — A progress bar shows how many frames have been processed in real time.
3. **Review the results** — Once complete you will see:
   - **Overall verdict** (AI / Uncertain / Human) with a confidence score.
   - **Frame grid** — every sampled frame with its individual AI score and colour-coded badge.
   - **Explanation panel** — click any frame to read the LLM forensic report.
   - **Top suspicious frames** — the three frames with the highest AI score are highlighted.
4. **Analyse another video** — Click the **FrameScope** logo in the header to reset and upload a new file.

## API Reference

The backend exposes a REST API on `http://localhost:8000`.

### `POST /api/detect`

Upload a video for analysis. Returns a `job_id` immediately; the analysis runs asynchronously.

**Request** (multipart/form-data):

| Field | Type | Default | Description |
|---|---|---|---|
| `file` | file | required | Video file to analyse |
| `sample_rate` | int | 30 | Sample one frame every N frames |
| `max_frames` | int | 40 | Maximum number of frames to analyse |

**Response:**

```json
{
  "job_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "status": "processing"
}
```

## Configuration

All backend settings are read from `framescope/.env` via Pydantic Settings.

| Variable | Required | Default | Description |
|---|---|---|---|
| `GROQ_API_KEY` | ✅ Yes | — | Groq API key for LLM explanations |
| `GROQ_MODEL` | No | `meta-llama/llama-4-scout-17b-16e-instruct` | Groq model used for frame explanations |
| `DEFAULT_SAMPLE_RATE` | No | `30` | Default frames to skip between samples |
| `DEFAULT_MAX_FRAMES` | No | `40` | Default maximum frames per video |

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---|---|---|
| `ValidationError: groq_api_key field required` | Missing or mislocated `.env` file | Ensure `framescope/.env` exists and contains `GROQ_API_KEY=...` |
| Backend starts but returns 500 on upload | Model not yet downloaded / OOM | Wait for model download to finish; try on a machine with more RAM |
| `CORS` error in browser | Frontend URL mismatch | Confirm the frontend runs on `http://localhost:5173`; update `allow_origins` in `main.py` if using a different port |
| Upload rejected with 400 | Unsupported file format | Use `.mp4`, `.mov`, `.avi`, `.webm`, or `.mkv` |
| Frames analysed but no explanations | Groq API error | Check your `GROQ_API_KEY` and Groq account rate limits |
| Very slow analysis | Running on CPU | Normal — expect ~5–15 s per frame on CPU. GPU dramatically speeds this up |
